# Common Build Problems

## Learning goals

- Recognize common Java build failures.
- Learn practical troubleshooting steps.
- Avoid committing generated output.

## Maven not installed

Symptom:

```text
mvn: command not found
```

Fix:

- install Maven; or
- use Maven Wrapper if the repository provides `mvnw` or `mvnw.cmd`.

## Wrong Java version

Symptom:

```text
release version 21 not supported
```

Fix:

- install the expected JDK;
- update `JAVA_HOME`;
- align CI Java version with local configuration.

## Dependency resolution failure

Possible causes:

- typo in dependency coordinates;
- unavailable version;
- temporary repository access problem;
- local Maven cache issue.

## `target` committed accidentally

The `target` folder is generated by Maven and should usually be ignored.

Add to `.gitignore`:

```gitignore
target/
```

## Tests pass locally but fail in CI

Common causes:

- local-only files;
- case-sensitive path differences;
- time zone differences;
- test order dependency;
- missing environment variables.

## Mini exercise

Create a troubleshooting checklist for a failing Maven build.

## Quick summary

Most build problems come from version mismatch, missing files, dependency issues, or generated output being treated like source.
