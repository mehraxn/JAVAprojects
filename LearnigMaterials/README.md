# Java Learning Materials

Beginner-to-intermediate Java study notes organized as a full learning path. The material starts with Java fundamentals and grows into backend topics such as architecture, workflows, persistence, build tools, data import, logging, and reports.

Most folders contain Markdown explanations, examples, common mistakes, and exercises. Some existing folders also include Java code examples.

## Start here

`00) START HERE` contains the study roadmap, file index, and cheat sheet. The numbered folders below are the main course sequence.

## Final folder structure

1) **Java Fundamentals**  
Learn JVM/JDK/JRE basics, variables, operators, strings, control flow, methods, and arrays.

2) **Basics OOP Core**  
Learn classes, objects, encapsulation, access modifiers, static/final, enums, records, inner classes, and core object methods.

3) **Inheritance and Polymorphism**  
Learn constructors, `super`, abstract classes, interfaces, overriding, overloading, dynamic dispatch, and UML basics.

4) **Software Architecture and Design Patterns**  
Learn how to organize larger Java applications using layers, facades, factories, repositories, services, and defensive DTOs.

5) **Generics**  
Learn generic classes, generic interfaces, iterators, wildcards, PECS, type erasure, and generic restrictions.

6) **Collections**  
Learn `List`, `Set`, `Map`, `Queue`, sorting, frequency counting, and common collection operations.

7) **Streams**  
Learn streams, lambdas, method references, filtering, mapping, reducing, collectors, grouping, partitioning, and `Optional`.

8) **Exceptions**  
Learn checked and unchecked exceptions, custom exceptions, exception dispatching, error handling, and try-with-resources.

9) **Application Layers, Domain Workflows, and Authorization**  
Learn service-layer workflows, validation strategy, custom business exceptions, role checks, audit fields, and workflow testing.

10) **Java IO**  
Learn readers, writers, NIO `Path` and `Files`, scanners, formatters, serialization, and resource handling.

11) **Testing with JUnit**  
Learn assertions, lifecycle methods, test structure, common testing patterns, and failure diagnosis.

12) **Git, Maven, Build Tools, and CI CD**  
Learn Git basics, Maven structure, `pom.xml`, dependencies, plugins, Maven Wrapper, compiler release, and GitHub Actions.

13) **JDBC and Databases**  
Learn JDBC, `PreparedStatement`, database connections, SQL execution, and transaction basics.

14) **ORM JPA and Hibernate**  
Learn ORM concepts, JPA entities, Hibernate, annotations, relationships, cascade, fetch types, and entity lifecycle basics.

15) **EntityManager, Persistence Units, and Transactions**  
Learn `EntityManager`, `EntityManagerFactory`, `persistence.xml`, persistence units, CRUD operations, transactions, rollback, and H2 testing.

16) **JPQL, TypedQuery, and Repository Pattern**  
Learn JPQL, entity-based queries, parameters, typed results, date range queries, aggregations, JPA repositories, and repository testing.

17) **Date and Time**  
Learn `LocalDate`, `LocalTime`, `LocalDateTime`, formatting, parsing, `Duration`, and `Period`.

18) **Data Import, Logging, and Statistical Reports**  
Learn CSV imports, row validation, date parsing, import result objects, logging, immutable reports, statistics, histograms, and outlier detection.

19) **Final Revision Pack**  
Review Java, OOP, collections, streams, exceptions, IO, testing, build tools, persistence, architecture, imports, logging, and reports.

20) **Exercises**  
Practice all topics with standalone exercises, including architecture, workflows, Maven/CI, JPA, JPQL, data import, logging, and reports.

## Suggested study method

1. Read the topic explanation.
2. Study the code example.
3. Review common mistakes.
4. Complete the mini exercises.
5. Use the final revision pack before interviews, exams, or project work.

## Notes about examples

Examples are generic Java learning examples using domains such as students, courses, products, orders, accounts, books, members, invoices, employees, and departments. They are intended for study and should be adapted carefully before use in real applications.
